# RulateAutoLiker.exe

В папке уже лежит Chromium (`ms-playwright`). Дополнительно ничего ставить не нужно.

## Как запустить

Открой PowerShell в этой папке и запусти:

```powershell
.\RulateAutoLiker.exe
```

## Первый запуск

1. Нажми `1. Первичная настройка + Telegram QR`.
2. Введи ID группы Telegram, например `-1004310824989`.
3. Если нужен топик, введи ID темы. Если весь чат, введи `0`.
4. Выбери действия для книги: лайк, оценки, закладка.
5. Для глав теперь два разных пункта:
   - `Открывать главы для просмотров` - главы будут открываться.
   - `Ставить лайк/спасибо главам` - будет нажиматься спасибо на главе.
6. Если хочешь только просмотры без спасибо: включи открытие глав, но выключи лайк главам.
7. `Сколько глав открывать на книгу`: `0` значит без ограничений.
8. Отсканируй Telegram QR.
9. Нажми `2. Войти в Rulate`, войди в аккаунт и нажми Enter.

## Основные пункты меню

- `3` - постоянная проверка Telegram.
- `Ctrl+C` - остановить постоянную проверку и вернуться в меню.
- `4` - один проход.
- `5` - только найти новые ссылки в Telegram.
- `6` - статистика.
- `8` - прокси только для Chromium.
- `9` - проверить, открывается ли Rulate с текущим прокси.

## Остановка на главах

Если нажать `Ctrl+C` во время обработки глав, программа запомнит текущую главу.

При следующем запуске она начнёт с этой же незавершённой главы, а потом пойдёт дальше как обычно.

## Платные главы

Если включено `Пропускать платные главы`, программа пропускает главы, у которых в таблице Rulate есть цена покупки/подписки.

Это работает и для аккаунта из команды перевода: даже если кнопка `читать` доступна, глава с ценой не считается бесплатной и не открывается.

## Один проход вручную

В пункте `4` можно просто нажать Enter - будет обычный один проход.

Можно вставить ID сообщений, диапазон или ссылки:

```text
6
6-20
6 8 10
https://t.me/c/4310824989/2/6
https://tl.rulate.ru/book/143612
```

Тогда программа принудительно обработает только эти книги. Если книга уже была обработана, статус для неё сбросится и она пройдёт заново.

## Повторная обработка

Обычный проход не кликает заново то, что уже записано в базе: лайк, оценки, закладку и просмотренные главы.

Если включена ежедневная перепроверка, программа каждый день возвращается к недавно найденным книгам и открывает только новые/ещё не отмеченные главы.

Принудительный запуск через пункт `4` со ссылкой, ID сообщения или диапазоном сбрасывает статус выбранной книги и прогоняет её заново.

## Ежедневная перепроверка

В пункте `1` есть настройки:

- `Сколько дней ежедневно перепроверять книги` - например `3`.
- `Лимит книг для ежедневной перепроверки в сутки` - например `3`.

Это нужно, чтобы в течение нескольких дней открывать новые главы у недавно найденных книг.

Если поставить `0`, настройка выключена или без лимита, как написано в вопросе.

## Если сайт, VPN или интернет упал

Программа делает несколько попыток открыть страницу. Если не получилось, пишет ошибку и не помечает книгу/главу как завершённую.

На следующем запуске она попробует снова.

## Если это твоя книга

Если Rulate не даёт поставить лайк/оценку на свою книгу, кнопки просто не найдутся, и программа пойдёт дальше. Главы для просмотра всё равно могут открываться, если включён просмотр глав.

## Аккаунты и твинки

Одна папка portable хранит один профиль Chromium и один аккаунт Rulate в `data\browser_profile`.

Программа не переключает несколько Rulate-аккаунтов внутри одной папки по очереди.

Если нужны разные аккаунты, сделай копии всей папки `rulate-auto-liker-portable` и в каждой один раз войди в свой аккаунт через пункт `2`.

## Где данные

- `config.json` - настройки.
- `data\telegram.session` - Telegram-сессия.
- `data\browser_profile` - аккаунт Rulate и куки Chromium.
- `data\rulate_auto_liker.sqlite3` - база книг, сообщений и глав.

Не удаляй `data`, если не хочешь заново входить в Telegram/Rulate и терять историю обработки.
